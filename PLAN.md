# 《AI 大模型不难》项目计划书（最终版）

> 周末读完的 LLM 面试通关书。28 课 + 2 附录，每课一个生活比喻 + 一个手算示例。
> 启动于 2026 年 7 月 15 日，同日完成 V1-V5 迭代。

## 定位

不讲代码，只讲你能在面试里说清楚的原理。竞品差异化：
- vs 图解大模型（Jay Alammar）：我们用比喻体系 + 手算示例，不是 300 幅图
- vs Happy-LLM（Datawhale）：我们不写代码，纯概念讲解
- vs 面试题库：我们给原理和机制，不只是 Q&A

## 最终结构（30 个文件）

```
Part 0  开篇（1 课）
Part 1  世界观（6 课）：Token→涌现→Prompt→解码策略→RLHF→量化蒸馏分布式
Part 2  引擎（8 课）：Transformer→Attention→多头→位置编码→残差→LayerNorm→Pre/Post-norm→Mask
Part 3  炼金（6 课）：参数vs数据→MLM/CLM→RLHF→DPO/GRPO→LoRA→训练硬核
Part 4  落地（6 课）：RAG→推理优化→Agent→MCP/A2A→前沿架构→多模态
Part 5  实战（2 课）：大厂真题上/下 + 系统设计 + 项目讲述
附录 A  NLP 基础速查
附录 B  理论速查表（显存/公式/默认值/模型对比）
```

## 一、为什么做这个项目

### 1.1 市场缺口

市面上的大模型教程分为三类，每类都有明确短板：

| 类型 | 代表 | 短板 |
|------|------|------|
| 面试八股合集 | 400-Interview-Questions, llm-interview-notes | Q&A 格式，零散的「标准答案」，没有故事线，看完记不住 |
| 从零实现教程 | LLMs-from-scratch, Happy-LLM | 代码驱动，适合工程背景读者，普通人卡在第一章 |
| 视频课程 | 吴恩达、李宏毅等 | 要 20+ 小时连续看，无法按需查阅，搜索定位困难 |

**我们做的是**：一本用「类比 + 图解 + 聊天感」把大模型核心概念说人话的开源书。对标姜伟生「鸢尾花书」的视觉密度和卡兹克的叙事温度。

### 1.2 用户画像

**核心读者**：想入行 AI 大模型方向但被数学和概念挡在门外的「好奇的普通人」。
- 有基础编程能力（会 Python），但没读过 Attention Is All You Need
- 面试前需要快速建立对 Transformer 全貌的直觉理解
- 更喜欢「先有感觉再看公式」而非「先推公式再找感觉」

### 1.3 差异化

- ❌ 不写 Q&A 合集（已有太多）
- ❌ 不写从零实现教程（LLMs-from-scratch 够好了）
- ✅ 写一个「有人味儿的故事线」——16 个概念串成一条完整的认知曲线
- ✅ 每个概念 = 一个类比 + 一张图解 + 一段聊天式讲解 + 两道思考题
- ✅ 不回避追问，章节自带「面试官可能怎么追问」模块

## 二、调研与借鉴

### 2.1 鸢尾花书（姜伟生 / Visualize-ML）

| 维度 | 借鉴什么 |
|------|---------|
| 视觉密度 | 每章配多张矢量图，用 Manim 生成动画 GIF 辅助理解 |
| 结构设计 | 分上下册，每章从「直觉」出发，再逐步引入公式 |
| 技术栈 | Jupyter Notebook + Python 代码 + 交互式图表 |
| 开源运营 | GitHub 仓库 + PDF 电子书 + 社区纠错机制 |
| 命名方式 | "不难"系列品牌效应强，"AI 大模型不难"可以借势 |

### 2.2 数字生命卡兹克（公众号）

| 维度 | 借鉴什么 |
|------|---------|
| 叙事节奏 | 具体的、当下的切入，不做宏大叙事开头 |
| 知识输出 | "聊着聊着顺手掏出来"，不是"下面我来科普" |
| 私人视角 | "我也踩过这个坑""我当时就愣住了" |
| 类比能力 | 每个抽象概念对应一个生活化比喻 |
| 文化升维 | 每章结尾从技术小点拉到更大的图景 |

### 2.3 GitHub 类似项目对比

| 项目 | Stars | 格式 | 与我们的差异 |
|------|-------|------|-------------|
| Visualize-ML/Linear-Algebra-Made-Easy | 1.4k | PDF + Jupyter | 数学领域，不是 AI |
| rasbt/LLMs-from-scratch | 38k+ | Jupyter | 代码实现向，本质是教程不是科普 |
| Datawhale/Happy-LLM | ~1k | Markdown | 偏实战，叙事感弱 |
| Lanqingsong/400-Interview-Questions | ~500 | Markdown | 纯 Q&A，信息密度低 |
| 我们的《AI 大模型不难》 | 目标 5k+ | HTML + PDF | 图解 + 类比 + 聊天感，填补"亲切科普"空白 |

**结论**：目前没有一本「用鸢尾花书的视觉密度 + 卡兹克的叙事温度」讲述大模型概念的开源书。这是我们的蓝海。

## 四、每课标准模板

```
【开头 - 感性切入】（200-400 字）
从一个具体的生活化场景/困惑开始，让读者觉得"对，我也遇到过"
不使用"在当今 AI 快速发展的时代"这类套话

【核心内容 - 图解 + 类比】（1500-2500 字）
- 主类比贯穿始终（如 Attention = 图书馆查资料）
- 至少一张手工绘制的 SVG 图解（标注关键节点）
- 公式出现时，先用人话翻译一遍，再展示数学符号
- 每一步都用「为什么要这么做」来驱动，而不是「接下来我们这样做」

【追问模块 - 面试官可能怎么问】（500-800 字）
- 3-5 个典型的面试追问
- 不是给出标准答案，而是给出「思考路径」
- 标注哪些是「加分金句」级别的回答

【思考题】（2 道）
- 1 道验证理解的（如"用你自己的话解释 X"）
- 1 道拓展思考的（如"如果去掉 Y 会怎么样"）

【文化升维 - 一碗鸡汤】（200-300 字）
从刚才的技术讨论自然过渡到一个更大的图景
不硬拔高，像聊着聊着突然感慨了一句
```

## 五、技术方案

### 5.1 写作工具链

```
Markdown 草稿 → Python (Mistune) 渲染 HTML → 静态站点
                                        → PDF (WeasyPrint)
                                        → SVG 图解内嵌
```

### 5.2 图解方案

- **轻量方案**（前期）：手写 SVG，嵌入 Markdown。我们在这个对话里已经验证过——每节课的图解都能在 5 分钟内生成干净的教学用 SVG
- **进阶方案**（后期）：Manim 动画 GIF，类似鸢尾花书的动画插图
- **代码示例**：纯 Python（PyTorch），不引入额外依赖，能跑起来就行

### 5.3 发布形式

| 渠道 | 格式 | 优先级 |
|------|------|--------|
| GitHub (moyai/ai-llm-made-easy) | Markdown + HTML | P0 |
| GitHub Pages | 静态网站 | P0 |
| PDF 电子书 | 排版后的 PDF | P1 |
| 公众号 / 知乎专栏 | 单篇拆发 | P2 |
