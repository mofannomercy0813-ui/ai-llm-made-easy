# 第 24 课：前沿架构，MoE、GQA、DeepSeek 怎么玩的

过去的 6 年里 Transformer 的内核没有变过，注意力、残差、归一化、位置编码。但外围架构成了一片狂野的实验场。2024 到 2025 年最沸腾的三个创新，MoE、GQA、还有 DeepSeek 的开源奇迹。

先说 MoE。Mixture of Experts。混合专家。

传统 Transformer 每一层的 FFN 是一个密集的大矩阵。MoE 把这个大矩阵拆成多个小矩阵，每个叫一个「专家」。一个 Token 来了之后，不是让所有专家都处理。而是用一个小型路由网络判断这个 Token 更需要哪些专家的帮助，选 Top-K（通常 K=2）。只有被选中的专家才激活计算。

效果是参数量暴增但计算量不暴增。DeepSeek-V3 有 1.6 万亿参数，但每次推理只激活 490 亿。参数量大所以表征能力强，激活量小所以推理成本低。完美兼顾。

MoE 训练难。负载均衡，有些专家会变成「网红」被大量 Token 选中，其他专家闲置。辅助损失可以缓解但增加了训练复杂度。通信开销，不同专家可能在不同 GPU 上，Token 需要在 GPU 之间路由。

再说 GQA。Grouped Query Attention。分组查询注意力。

标准的 Multi-Head Attention 中，Q、K、V 有相同数量的头。MQA（Multi-Query Attention）把 K 和 V 压成 1 个头，Q 保持多头。参数量骤降但 K 和 V 的信息瓶颈可能损伤效果。GQA 是一种中间方案，K 和 V 分成 G 组，Q 保持全头。组内的 Q 共享同一组 K 和 V。Llama 2 和 Qwen 用的就是 GQA。在精度和速度之间找了一个漂亮的折中。

SwiGLU。一个激活函数。原来 FFN 用 ReLU 做非线性，负数直接砍零。SwiGLU 用 Swish 激活的门控单元，平滑过渡，信息保留更多。Llama 引入后基本上所有新一代模型都跟进了。属于那种「效果有提升但你不做也不会死」的微优化。

DeepSeek-V3 把这些技术全整合了。MoE 做参数规模。GQA 做推理加速。还有 Multi-head Latent Attention，把 KV Cache 压到一个极小的潜在空间里，显存占用大幅压缩。它能在 128K Token 的长序列上流畅推理，同时保持开源模型里顶尖的综合能力。这是工程系统设计的教科书级案例。

安全性。Llama Guard 和 Prompt Guard 是做输入输出过滤。模型本身也有 RLHF 的对齐约束，拒绝回答武器、毒品、色情等敏感话题。但越狱攻击始终存在，用巧妙的 Prompt 绕过了安全护栏。攻防在持续迭代。

## 追问模块

**追问 1：「MoE 的路由网络是怎么训练的？」**

跟主模型一起训。路由网络（也叫门控网络）是一个小型的线性层加 Softmax。它的梯度通过 Top-K 选择的 Token 回传。辅助负载均衡损失确保专家使用率均衡。

**追问 2：「GQA 相比 MHA 差多少精度？」**

大多数任务上精度损失感知不到，1% 到 2% 的 benchmark 差异。但推理显存（KV Cache 部分）降低倍数等于 G 值。Llama 2 70B 用 GQA(G=8) 相比 MHA(头数 64) 把 KV Cache 压到 1/8。

**追问 3：「DeepSeek-V3 的 MLA 是什么原理？」**

MLA 把 KV 投影到一个比 d_head 更小的潜在维度，从潜在空间再恢复完整 KV。相当于给 KV 做了有监督的有损压缩。128K 长序列推理时 KV Cache 可降至传统方案的 1/10。这是 DeepSeek 长上下文推理便宜的核心原因。

## 思考题

1. MoE 的专家调度像不像你公司里的部门分派，有些部门常年满负荷，有些部门闲着。你有什么管理学的办法解决这个问题。MoE 的解决方案跟你的一样吗。

2. 如果 GQA 的 G 越大（越接近 MHA），精度越高但越慢。G 越小（越接近 MQA），速度越快但精度越低。给你一个新模型，你怎么定这个 G 值。
