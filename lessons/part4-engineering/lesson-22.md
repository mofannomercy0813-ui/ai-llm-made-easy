# 第 22 课：AI Agent，让模型会干活

你让 ChatGPT 帮你写周报。它写了。

你让它帮你发周报。它说，抱歉，我是一个语言模型，我无法执行操作。

这就是「回答问题」和「完成任务的本质区别。Agent 把模型从「告诉你该怎么做」变成了「直接帮你做完」。它不只是生成文本。它能感知环境、做决策、执行动作、观察结果、然后根据结果继续调整下一步。

一个完整的 Agent 有四件事。感知，知道现在是什么情况，输入了什么指令、拿到了什么返回结果、当前的上下文是什么。决策，基于感知到的信息决定下一步做什么，是继续执行还是换一个工具。执行，调用相应的工具，浏览器、计算器、搜索引擎、数据库、任何 API。学习，从执行结果中提取经验，更新自己的决策策略。

感知才可能决策。决策才可能执行。执行后才会学习。四个环节轮流转。Agent 的循环周期是 P-D-E-O 观察-决策-执行-反馈。

但一个纯 LLM Agent 有天然短板。它不会打电话，不会浏览网页，不会操作你的 Excel。它唯一会的交互方式就是生成文本。

这就是 Function Calling 的意义。Function Call 给了模型一个「呼叫外部工具的按钮」。模型在生成回答的过程中，不说普通文本，而是输出一段结构化的 JSON，{function, "search_web", arguments, {query, "今天北京天气"}}。外部的 Agent 框架截获这段 JSON，执行真实的函数调用，去搜索引擎里请求结果，把结果返回给模型。模型再基于结果继续生成最终的文本回答。

单 Agent 够用吗。不一定。多 Agent 系统（MAS）把多个各司其职的 Agent 组合成一个协同系统。一个 Agent 负责检索，一个负责分析，一个负责写报告。各干各的，信息在 Agent 之间流转。

多 Agent 的复杂度是指数级的。Agent 之间的通信协议怎么设计。任务怎么分配。出错了谁来兜底。

反应式 Agent 最简单，看到什么就反应什么，没有记忆，没有规划。对于简单的分类和判断够用。慎思式 Agent 会做规划和推理，维护内部状态，跟踪目标进展。混合式 Agent 结合两者，大部分时候快速反应，碰到复杂的决策时暂停、调用规划能力、做完再继续反应。

## 追问模块

**追问 1：「Agent 和普通 Chatbot 到底差在哪？」**

Chatbot 是单轮对话，你问它答。Agent 是多轮多步骤的任务执行，它能调用工具、观察结果、规划下一步。Chatbot 是被动响应，Agent 是主动完成任务。

**追问 2：「Function Calling 和 API 调用有什么区别？」**

Function Calling 是模型输出的结构化指令，模型决定「该调哪个函数」「传什么参数」。然后外部的框架去执行这个调用并返回结果。API 调用是程序员的活，固定逻辑。Function Calling 是模型自己的决策。

**追问 3：「多 Agent 系统最大的坑是什么？」**

通信开销。Agent 之间来回传递信息，每多一个 Agent 就多一堆通信链路。协调出错时会出现死循环，两个 Agent 互相等对方的结果。容错和超时机制必须从第一天就设计好。
