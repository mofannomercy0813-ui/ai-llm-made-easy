# 第 16 课：RLHF 全流程，SFT → RM → PPO

第四课我们用脱口秀彩排讲了一遍 RLHF 的全貌。现在是拆开工程细节的时候了。

第一步。SFT。Supervised Fine-Tuning。

给你一堆高质量的人类标注数据。每条数据是一个 prompt 和一个被人类认为「完美」的回答。你要让模型的输出尽可能接近这些完美回答。怎么做。标准的交叉熵损失训练。跟预训练一模一样，只是数据的质量和方向性完全变了。预训练的语料是全网文本。SFT 的语料是几千到几万条人类精心编写的示例。

SFT 完了之后模型已经能说出像样的话了。但它的回答风格是「模仿人类标注员的平均水平」。标注员说什么风格的话，模型就学什么风格的话。这还不够。

你需要一个自动化的质量判断机制。这就是 RM。

第二步。RM。Reward Model。

RM 是一个独立训练的小模型。给一段 prompt 和一个 response，输出一个分数，这个回答有多好。RM 的训练数据怎么来。给同一个 prompt，让 SFT 模型生成好几个不同的回答（比如 4 个）。人类标注员对这 4 个回答排序，A 比 B 好，B 跟 C 差不多，C 比 D 好。RM 的损失函数叫 pairwise ranking loss，让模型学会输出一个标量奖励 r，使得人类喜欢的回答的 r 更高。

为什么要排序而不是直接打分。因为打分的尺度太主观了。张三觉得这个回答 7.5 分，李四觉得 8.5 分。但排序是稳定的，所有标注员都能判断 A 比 B 好。

现在你有了一个自动打分器。它可以告诉你任何回答的质量高低。

第三步。PPO。Proximal Policy Optimization。

这是整个 RLHF 里最复杂的部分。你有一个策略模型（就是 SFT 之后的模型）。你有一个奖励模型（就是 RM）。你给策略模型一组 prompt，让它生成回答。奖励模型对这些回答打分。你再根据分数更新策略模型的参数，让高分回答的生成概率变高。

PPO 的核心是 Clip 机制。如果新旧策略的概率比超过了 [1-ε, 1+ε] 的范围，梯度就被截断。这防止单步更新过大导致训练崩溃。它是 PPO 名字里 Proximal（近端的）的来源，保持新策略离旧策略不远。

PPO 还有一个关键的约束，KL 散度惩罚。让新策略的输出分布不要偏离原始的 SFT 模型太远。如果没有这个约束，策略模型可能会走火入魔，为了拿高分不断生成 RM 喜欢的但完全失去语言质量的回答。这叫 Reward Hacking，模型学会了怎么讨好评委而不是真正提升回答质量。KL 约束就是给模型划了一条不能超过的红线。

PPO 训练需要同时维护四个模型，策略模型（主训练对象）、参考模型（冻结的 SFT 模型，计算 KL）、奖励模型（RM）、价值模型（辅助 PPO 的优势函数计算）。光是显存需求就比普通训练大好几倍。

这就是为什么真正做完整 RLHF 的门槛这么高。

## 追问模块

**追问 1：「RM 为什么不直接用人类标注打分而要排序？」**

排序是稳定的。打分是主观的。一个标注员可能觉得所有回答都很好全部打 9 分，另一个可能觉得都一般全部打 5 分。这两个标注员产生的是完全不同的训练数据。但排序的结果是一致的，A>B>C 这个顺序在所有人眼里都一样。排序的数据质量和复用率远超打分。

**追问 2：「PPO 能不能直接用一个现成的打分模型替代 RM？」**

能。但专用 RM 的效果通常更好。因为专用 RM 是用你的 SFT 模型生成的回答微调过的，它更懂你的模型可能出什么类型的问题。用别人训的 RM 可能会产生误判。但 GRPO 选择了一条不同的路，不用 RM 而是用一组回答的相对分数来做优化。下节课聊 GRPO。
