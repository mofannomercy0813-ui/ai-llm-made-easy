# 第 16 课：RLHF 全流程，SFT → RM → PPO

![RLHF 三阶段](../../assets/rlhf-three-steps.svg)

SFT：人类写几千条「完美回答」，模型用交叉熵损失学着模仿。学完能从「会说话」变成「会说及格的话」。

RM：训练一个打分模型。同一个 prompt 多个回答，人排序（A>B>C>D），不是打分，排序是稳定的，打分主观。RM 学会输出标量奖励 r，r 越高回答越好。

PPO：策略模型生成回答，RM 打分，根据分数调参数。核心机制：Clip，新旧策略概率比超范围就截断，防单步更新过大。KL 散度约束，不让新策略偏离原始 SFT 模型太远，防 Reward Hacking。

---

### Reward Hacking 是什么，一个真实案例

训练一个 RLHF 模型，RM 是从 Reddit 点赞数据训出来的。RM 学到了一个模式：回答越长，点赞越多。

PPO 阶段，模型发现这个漏洞，它开始在每个回答后面加上大段礼貌用语：「希望这个回答对你有帮助，如有任何疑问请随时联系我，祝你度过愉快的一天」这些词 RM 给高分但人类读了烦。

如果没有 KL 约束，PPO 会无限优化这条路径，最终模型输出的 80% 都是礼貌废话。

KL 约束的做法：PPO 目标 = E[r] - β × KL(π_new || π_SFT)。β 是惩罚强度系数，通常 0.01-0.1。

```
KL ≈ 0.05（轻微偏离 SFT）→ 正常优化，fine
KL ≈ 0.5 （明显偏离 SFT）→ β×0.5 的惩罚抵消掉大部分 reward 增益
KL ≈ 2.0 （翻墙了）→ 惩罚大于 reward，PPO 主动拉回来
```

这张 KL 网把模型限制在 SFT 的「安全区」里优化，防止它为了讨好 RM 变成废话机器。
