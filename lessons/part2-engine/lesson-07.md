# 第 7 课：Attention 公式，手写一遍就懂了

Attention 像在图书馆查资料。Q 是你的问题，K 是每本书的标签，V 是书的内容。

公式四步。

第一步，Q × K^T。计算你的问题和每本书标签的相似度。结果是 n × n 矩阵，每个 token 对每个 token 的相关性分数。这就是 O(n²) 的来源。

第二步，除以 sqrt(d_k)。Q 和 K 的维度大了点积值会爆炸，送进 softmax 后梯度消失。除以 sqrt(d_k) 把数值压回稳定区间。不除的话训练会发散。

第三步，softmax。把分数变成概率，每一行加起来等于 1。e^x 让高分更突出、低分接近零。但 e^0 = 1，所以即使两个词完全正交，注意力也不会是零，每个词都能分到一点关注。

第四步，× V。按 softmax 算出的权重对 V 做加权求和。「猫坐在垫子上」里，「坐」的输出 = V_坐 × 0.5 + V_猫 × 0.3 + V_垫子 × 0.15 + V_上 × 0.05。它不是「坐」的原始含义，而是融入了上下文的新表示。

Attention(Q, K, V) = softmax(QK^T / sqrt(d_k)) V

就这一行。四行 PyTorch 写完。

```python
def attention(Q, K, V):
    d_k = Q.size(-1)
    scores = Q @ K.transpose(-2, -1) / (d_k ** 0.5)
    return F.softmax(scores, dim=-1) @ V
```

## 追问模块

**追问 1：「Q 和 K 为什么分两个矩阵？」** 语言不是对称的。「猫」对「坐」的关注和「坐」对「猫」的关注不一样。分两个矩阵给了不对称的表达能力。

**追问 2：「除以 sqrt(d_k) 而不是别的？」** 点积的方差跟 d_k 成正比。除以 sqrt(d_k) 恰好让 softmax 输入方差为 1。

**追问 3：「FlashAttention 怎么解决 O(n²)？」** 不把完整的 n × n 矩阵写进 HBM。分块算，online softmax 保证数值正确。第 21 课详讲。
