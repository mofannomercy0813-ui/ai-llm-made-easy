# 第 7 课：Attention 公式，手写一遍就懂了

上一课说 Transformer 的核心是注意力。注意力是什么。

先忘掉公式。想象一个场景。

你走进一座巨型图书馆。你想找关于「如何烤面包」的资料。图书馆里有一百万本书，每本书的封面上写了标题和关键词。书里才是你要的内容。

你会怎么做。

你不会把一百万本书全翻一遍吧。你会怎么做。你会拿你的问题，「如何烤面包」，去跟每一本书的标题做匹配。标题里有「面包」「烘焙」「烤箱」这些词的书，你多分一些注意力。标题里是「宇宙起源」「Python 编程」「明朝历史」的，你少分注意力。然后你把所有书的内容，按你分配的注意力权重，做一个加权混合。

这就是 Attention。

只不过模型在千分之一秒内完成了这一百万次的匹配和混合。

现在看公式。它就是图书馆流程的数学表达。

Attention(Q, K, V) = softmax(Q × K^T / sqrt(d_k)) × V

拆成四步。

第一步。Q × K^T。

Q 是你的「查询」，你在图书馆里找什么。K 是每本书的「标签」，这本书是关于什么的。Q 乘以 K 的转置，就是在计算你的问题和每本书标题的相似度。相似度越高，分数越高。

在 Transformer 里，Q 和 K 都是从同一个输入向量投影出来的。每个词同时扮演「查询者」（Q）和「被查询标签」（K）两个角色。当你处理句子「猫坐在垫子上」的时候，位置「坐」的 Q 向量会去查询每个位置的 K 向量，发现「猫」和「垫子」跟它相关，「上」也相关但弱一些。

结果是 n × n 的相关性矩阵。你知道为什么是 O(n²) 复杂度了。

第二步。除以 sqrt(d_k)。

为什么。因为 Q 和 K 的维度 d_k 可能很大（比如 64 或 128）。两个随机向量的点积期望值随维度增长。如果 d_k 很大，点积值就会很大。点积值太大送进 softmax 会出问题，softmax 会输出极端值，梯度消失。

除以 sqrt(d_k) 就是把数值压回一个合理区间。原论文作者发现不除的话训练会发散。一个细节改变生死。

第三步。softmax。

softmax 做两件事。把分数变成概率，每一行的所有值加起来等于 1。并且 e^x 的特性会让高分更突出、低分更接近零，但不是零。

这个「不是零」极其重要。即使两个词在 QK 空间里完全正交、点积为 0，softmax 给它的概率也不是 0。因为 e^0 = 1。所以每个词至少能分到一丁点注意力。

这就是为什么「猫坐在垫子上」里，即使「猫」和「上」在语义上不太相关，「猫」仍然能分到「上」的一小点注意力。

第四步。× V。

V 是书的「内容」，这本书到底说了什么。Attention 的最终输出就是把 V 矩阵里的每一行，按照 softmax 算出来的注意力权重，做加权求和。

处理「坐」这个词的时候，V_坐 × 0.5 + V_猫 × 0.3 + V_垫子 × 0.15 + V_上 × 0.05。结果就是这个位置的最终表示。

它不是「坐」的原始含义。它是「坐」融入了「猫」和「垫子」之后的新含义。

上下文。就是这么产生的。

## 追问模块

**追问 1：「Q 和 K 为什么不能用同一个矩阵？」**

如果用同一个矩阵，QK^T 就变成了 K 跟自己的对称积。对称矩阵意味着注意力也是对称的，token A 对 token B 的关注度和 B 对 A 一模一样。但语言不是对称的。「猫」对「坐」的关注，和「坐」对「猫」的关注，是不一样的。分开 Q 和 K 给了模型不对称的表达能力。

**追问 2：「为什么要除以 sqrt(d_k) 而不是 sqrt(d_model) 或者别的？」**

因为每头注意力的维度是 d_k，点积的方差跟 d_k 成正比。除以 sqrt(d_k) 恰好让点积的方差等于 1，保证 softmax 的输入在稳定区间。这是一个方差归一化的操作。

**追问 3：「你能手写一个极简版 Attention 的 PyTorch 代码吗？」**

```python
import torch
import torch.nn.functional as F

def attention(Q, K, V):
    d_k = Q.size(-1)
    scores = Q @ K.transpose(-2, -1) / torch.sqrt(torch.tensor(d_k))
    weights = F.softmax(scores, dim=-1)
    return weights @ V
```

就四行。整个现代 AI 世界的基石就这四行。

**追问 4：「Seq_len 128K 的时候 n² 是多少？怎么解决？」**

大约 160 亿次运算。解决方案是 FlashAttention，不在 HBM 里存完整的 n×n 矩阵，分块计算，用 online softmax 保证数值正确。第 21 课会详细讲它。
