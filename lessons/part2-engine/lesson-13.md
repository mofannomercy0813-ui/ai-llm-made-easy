# 第 13 课：两种 Mask，为什么模型必须戴眼镜

你拿到一张卷子。上面印着十道题。最后两行是阅卷官的批注草稿，上面乱涂了一堆数字，毫无意义。

你会去看那两行吗。你不会。你看的那几秒就是你注意力被浪费的几秒。

现在想象你是一个正在考试的学生。你做到第五题。正确答案就在旁边的试卷背面印着呢。不遮住的话你会不会偷看。你肯定会被诱惑，你的注意力不可避免地被那些不该看的答案分走。

这就是 Transformer 需要的两种 Mask。

Padding Mask 遮住的是那些无意义的填充位。

训练的时候一个 batch 里句子不等长。「我爱 NLP」只有三个 token，「你今天过得怎么样」有七个 token。要组成一个整齐的矩阵，短的句子就得补上 PAD 填充符。这些 PAD 位在 embedding 层是随机向量，不包含任何语义信息。

如果不加 Padding Mask，真实 token 的注意力会被 PAD 分走一部分。Softmax 的和是 1，分给 PAD 的权重就是抢了真实 token 的注意力预算。PAD 位也参与 Loss 计算，模型会试图预测 PAD 的下一个词，产生无效的梯度更新。

所以 Padding Mask 在注意力矩阵里把 PAD 对应的行和列设为可屏蔽状态。这些位置的数值被替换成负无穷大。过 Softmax 后 e^{-∞}=0，权重归零。彻底从计算中消失。

Encoder 需要。Decoder 也需要。PAD 不是语义，无论谁都不能看它。

Sequence Mask 遮住的是未来。

Decoder 训练时用的是 Teacher Forcing，把完整的正确答案一次性喂进去，但有因果限制。训练时处理第三个 token，模型理论上可以看到第四、第五、第六个 token，这些都是未来的信息。但推理时你看不到未来。如果训练时让模型依赖未来的信息，推理时它会因为看不到未来而崩溃。

这种训练和推理的输入不一致叫曝光偏差。

Sequence Mask 就是一个下三角矩阵。对角线及以下保留，过去和自己可见。对角线以上全部屏蔽，未来不可见。跟 Padding Mask 一样的实现，需要屏蔽的位置设为负无穷，Softmax 后归零。

Decoder 里的每一个 Self-Attention 层都戴着这块下三角眼镜。

Encoder 不需要。因为 Encoder 是双向理解，它可以同时看见过去和未来。这也是为什么 Encoder 适合理解任务，给它整句话，它能同时看到前后文。

## 追问模块

**追问 1：「为什么掩码值是负无穷而不是 0？」**

e^0 = 1。如果用 0，被屏蔽的 token 仍然会分到权重，softmax 后占比大概是 1/(e²+e¹+e^0+e^{-1}+...)。只有 e^{-∞}=0，才能让权重精确为零。实践中用 -1e9 或 float('-inf')。

**追问 2：「Sequence Mask 在推理时也起作用吗？」**

必须起作用。推理是自回归的，一次生成一个 token，已有序列逐位增长。每生成一个新 token，它可以看到前面所有已生成的 token，但不能看到还没生成的。Mask 随着序列长度动态扩大，训练时 Mask 是正方形的，推理时 Mask 是正方形的但每次尺寸都在变。

**追问 3：「Encoder 有没有 Sequence Mask？」**

没有。Encoder 是双向注意力。它需要的是同时看到前后的完整信息来「理解」这句话。这也是为什么你问 BERT「今天天气怎么样」时，它不需要从左到右逐字读，它一眼看了整句。
