# 第 9 课：位置编码，模型怎么知道词顺序

Attention 对词序不敏感，「猫追老鼠」和「老鼠追猫」的 QK^T 结果一样。必须注入位置信息。

绝对位置编码：每位置分配固定向量，加到 embedding 上。原版 Transformer 用 sin/cos，不同频率捕捉不同距离的关系。致命伤是外推差，训练时最大 512，推理给 1024 直接被陌生编码搞懵。

相对位置编码：不关心你在第几位，关心离我多远。距离 10 就是 10，跟起点位置无关。学习一个「距离→偏置」的映射表，加到 Attention 分数上。外推比绝对强，但有最大距离限制。

RoPE：当前最优解。把 Q 和 K 向量每两维当作 2D 箭头，根据位置旋转。位置 m 转 m×θ 度。不同频率的维度转得不一样快，高频捕捉局部，低频捕捉全局。旋转后做内积：Q_m·K_n = q·(R_{n-m}k)，只依赖相对位置 n-m。Llama、Qwen、DeepSeek 全部用它。

### RoPE 怎么工作的，两个词走一遍

句子只有两个词：「猫」（位置 0）、「坐」（位置 1）。d=4，θ=[1.0, 0.5]。

**Q_猫 = [0.8, 0.6, 0.3, 0.9]**

**位置 0 的旋转**：
- 维度 (0,1)：没有旋转，角度 = 0×1.0 = 0 → [0.8, 0.6] 不变
- 维度 (2,3)：没有旋转，角度 = 0×0.5 = 0 → [0.3, 0.9] 不变

**Q_坐 = [0.4, 0.2, 0.7, 0.5]**

**位置 1 的旋转**：
- 维度 (0,1)：旋转 1×1.0 = 1 弧度。cos(1)≈0.54, sin(1)≈0.84
  - 新值：[0.4×0.54 - 0.2×0.84, 0.4×0.84 + 0.2×0.54] = [0.05, 0.44]
- 维度 (2,3)：旋转 1×0.5 = 0.5 弧度。cos(0.5)≈0.88, sin(0.5)≈0.48
  - 新值：[0.7×0.88 - 0.5×0.48, 0.7×0.48 + 0.5×0.88] = [0.38, 0.78]

现在 Q_猫·K_坐，两个向量做内积。因为 K_坐 也经过了同样的旋转，内积结果里自动包含了一个「旋转角 = 1-0 = 1 的相对位置信息」。如果句子变长到位置 100 和 101，旋转角还是 1，结果完全相同。

这就是 RoPE 外推强的数学原因：不管句子多长，模型只受相对位置差影响。
