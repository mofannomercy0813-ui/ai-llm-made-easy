# 第 25 课：大厂真题精讲（上）

前面的 24 课你在学知识。这两课你回到面试现场。

不是给你标准答案让你背。是把面试官的追问路径还原出来，他先问什么，你答了什么之后他接着问什么，从哪个方向追问，问多深算深。你跟着这个路径走，就知道自己缺什么。

记住一条铁律。大部分面试官不是故意为难你。他问的是他日常工作中真的会碰到的问题。你如果能用自己的话把他的问题讲清楚，他就服。

五道真题。

---

字节跳动。PEFT 方案你怎么选。

不是让你背 LoRA 的定义。他问的是决策能力。你怎么选。基于什么因素。

标准回答先给结论。通用场景首选 LoRA。显存极度受限时选 QLoRA。精度要求极高且不在乎推理开销时选 Adapter。简单分类任务能用 Prompt Tuning 就不用 LoRA。

追问。你说 r=8 到 16，怎么知道该选 8 还是 16。看任务复杂度。任务越复杂需要越高的秩。简单的情感分类 r=4 甚至 r=2 都够。复杂推理任务需要 r=16 甚至 r=32。另外可以观察训练 Loss，如果 Loss 不降了但还没到你的期望水平，试着加 r。

追问 2。你把 QLoRA 和 LoRA 的区别给我讲透。QLoRA 就是 LoRA 加 NF4 量化，基座模型被压到 4 bit，LoRA 适配器保持 FP16 训练。NF4 是一种针对正态分布数据设计的 4 bit 格式，比普通的 INT4 对神经网络权重的表达更精准。还有双重量化，连量化的 scale 和 zero-point 参数本身也被量化了，进一步省显存。

追问 3。你实际落过地吗，碰到什么坑。诚实说。LoRA 的 alpha 和 r 的比例出问题会让训练不稳定。alpha/r 太小适配器效果太弱，太大适配器压过原始模型导致原来的能力丢失。QLoRA 的显存占用不是完全可预测的，实际占用往往比理论值高 10% 到 20%，因为量化参数、缓存、临时张量也占地方。

---

腾讯。RAG 系统你怎么设计。

这不是一个技术选型题。这是一个系统设计题。你从切分开始，一路讲到生成，再讲评估，再讲坑。

标准回答。五步。文档切分（200 到 500 Token 每块），embedding 索引（选 BGE 或 text-embedding-3），向量检索加关键词混合召（BM25 做精确匹配），重排序（Cross-Encoder 重排前 K 个候选），生成（拼 Prompt 喂 LLM）。评估用 RAGAS，检索召回率、答案忠实度、端到端相关性三个维度分别打分。

追问。chunk size 你是怎么试出来的。从 300 开始，跑一批典型问题，看检索召回率，被正确检索的文档占总相关文档的比例。太小了召回碎片化，一个完整的段落被切成了三块，只能检索到其中一块。太大了召回模糊，500 字的文档块包含了太多无关信息。一般在 200 到 500 之间反复试。不同文档类型（新闻、技术文档、聊天记录）的最优 chunk size 都不一样。

追问 2。向量数据库你打算用哪个，为什么。ChromaDB 做原型最快，零配置。Milvus 做生产，支持分布式、十亿级向量、混合检索。Qdrant 介于两者之间。不是哪个好哪个不好，是你的阶段需求决定的。

---

阿里巴巴。偏差和方差怎么把握。

经典但容易答乱。偏差高是欠拟合，训练误差高。方差高是过拟合，训练误差低测试误差高。大模型里面偏差和方差不是很好分析，因为训练一次太贵了没法跑几十次取方差。

标准回答。偏差高就继续训，加数据，加参数，加训练步数。方差高就做正则化，Dropout，权重衰减，早停。PEFT 天然低方差，只训 0.1% 的参数很难过拟合。全参微调小心，数据少的时候方差极高。

追问。Debug 的时候你怎么判断当前是偏差问题还是方差问题。看 Loss 曲线。训练 Loss 和验证 Loss 的 gap 小但都高，偏差。gap 大，方差。如果 gap 越来越大，马上停，加正则化。

---

美团。Agent 你怎么拆。

又是一个系统设计题。标准回答往四个维度拆。感知层，怎么获取环境信息（用户输入、工具返回、上下文）。决策层，基于感知信息选择行动（ReAct 框架，思考-行动-观察循环）。执行层，把决策变成真实的函数调用。记忆层，短期记忆（当前对话上下文），长期记忆（用户偏好、历史对话）。

追问。单 Agent 和多 Agent 怎么选。任务边界清晰、可拆解的任务用多 Agent，一个负责检索一个负责分析。任务简单、不需要分工协作用单 Agent，减少通信开销和出错概率。

追问 2。Agent 调用了错误的工具怎么办。错误处理的三层防御。第一层工具参数校验，入口处检查。第二层异常重试，失败后最多重试 N 次，换参数或换工具。第三层回退机制，多次失败后停止循环，给用户一个兜底回答。

---

小红书。MoE 怎么玩更稳。

标准回答。MoE 训练三个核心坑。负载均衡，网红专家和闲置专家。解法是加辅助负载均衡损失。通信开销，专家在不同 GPU 之间路由 Token。解法是 All-to-All 通信优化。训练不稳定，不同专家收到不同 Token 导致训练内的梯度骤变。解法是更保守的学习率搭配更频繁的负载均衡检查。
