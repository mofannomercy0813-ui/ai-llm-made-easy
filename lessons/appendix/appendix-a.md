# 附录 A：NLP 基础速查

本书假设你有一定的机器学习基础。如果下面这些概念你看起来有些吃力，这一页是给你的。

### Embedding — 高维空间里的语义坐标

计算机只能算数字。把文字变成数字的过程叫 Embedding。

最简单的做法是 One-Hot：10 万个词，每个词用一个 10 万维的向量表示，只有自己那一位是 1，其余全是 0。维度爆炸，且「猫」和「狗」的距离跟「猫」和「汽车」一样远，没有语义信息。

Word2Vec（2013）改变了这一点。不是人为定义向量，而是让模型从大量文本里自己学。最终「猫」和「狗」的向量夹角很小，「猫」和「汽车」的夹角很大。Embedding 的本质：语义相近的词在高维空间里距离近。

Transformer 用的 Embedding 更高维（768 或 4096），训练方式更复杂，但核心思想没变。

### 损失函数 — 衡量「猜得多离谱」

模型输出一个概率分布，正确答案是第 42 个 Token。怎么算出模型「猜得多错」？用损失函数。

交叉熵损失（Cross-Entropy Loss）是 LLM 训练的唯一损失函数：
$$L = -\log(p_{\text{correct}})$$
如果模型给正确答案的概率是 0.9，Loss = -log(0.9) ≈ 0.105。如果概率是 0.01，Loss = -log(0.01) ≈ 4.6。模型越确信正确答案、Loss 越低。

KL 散度度量两个概率分布的差异。在 RLHF 里用来约束新策略不要偏离 SFT 太远（第 16 课）。

### Softmax — 把任意实数变成概率分布

神经网络的最后一层输出是一个实数向量 [2.1, -0.5, 3.7]。这些数叫 logits，范围是 (-∞, +∞)，不能当概率用。

Softmax 两步：$e^{x}$ 把所有值变成正数，除以总和让它们加起来等于 1。
$$\text{softmax}(x_i) = \frac{e^{x_i}}{\sum_j e^{x_j}}$$
[2.1, -0.5, 3.7] → [0.14, 0.01, 0.85]。

Softmax 的特性：$e^x$ 是指数函数，会放大高值和低值的差距。3.7 和 2.1 的差是 1.6，但 $e^{3.7} ≈ 40.4$、$e^{2.1} ≈ 8.2$，差距被放大了 5 倍。这就是为什么你经常听到「softmax 让高分更突出」，它不是线性缩放，是指数放大。

### 激活函数 — 给线性层加一些弯曲

没有激活函数，无论堆多少层 Transformer，整个模型只是一个巨大的线性变换，只能学到线性关系。激活函数引入非线性，让模型能拟合更复杂的模式。

Sigmoid 把输入压到 (0,1)，用于门控（LSTM 的门）。现在的模型基本不用了，因为它有饱和区梯度消失的问题。

ReLU（max(0, x)）简单粗暴，CV 领域标配。负数端梯度为 0，可能导致「神经元死亡」。

GELU 是 ReLU 的平滑版，加了高斯分布的修正。Transformer 和 LLM 的首选激活函数。Swish / SiLU 也很常见。

### 常见的超参与

epoch — 把全部训练数据完整过一遍叫一个 epoch。LLM 预训练通常 1-3 个 epoch（数据太多，来不及多轮）。

batch size — 每次梯度更新用多少条数据。大 batch 梯度稳定但吃显存，小 batch 迭代快但梯度波动大。LLM 预训练的 batch size 通常在百万 Token 级别。

learning rate — 每次梯度更新的步长。太大损失震荡甚至发散，太小学得太慢。典型值 1e-4 到 5e-5。配合 warmup（前几千步慢慢加）和余弦衰减（后段慢慢减）。

warmup — 训练初期学习率从 0 线性增长到目标值。Post-norm 时代必须，Pre-norm 后重要性下降但仍是标配保险措施。
